## Технический стек сайта

### Архитектура
- **Single-file SPA** — весь HTML, CSS, JS в одном файле `index.html`
- **Табовая навигация** — `switchTab()` переключает `data-tab` ↔ `id="tab-{name}"`
- **Canvas-анимация** — частицы с хаотичным движением и редкими временными связями
- **Без фреймворков** — чистый HTML/CSS/JS, Google Fonts (Inter + Space Grotesk)

### Вкладки (tabs)
| data-tab | id | Описание |
|---|---|---|
| `home` | `tab-home` | Главная (герой + features) |
| `services` | `tab-services` | Услуги (6 карточек pkg-grid) |
| `tech` | `tab-tech` | Технологии (tech-grid + arch) |
| `business` | `tab-business` | Бизнесу / B2B (biz-grid + biz-why) |
| `investors` | `tab-investors` | Инвесторам (inv-grid) |
| `about` | `tab-about` | О нас + DAO-блок |
| `join` | `tab-join` | Присоединиться (форма) |

### CSS-переменные
```css
--bg:#07070d  --bg2:#0d0d16  --card:rgba(255,255,255,0.03)
--blue:#4f7df5  --purple:#6c5ce7  --gold:#d4a853  --cyan:#22d3ee  --green:#4ade80
--t1:#ffffff  --t2:#8a8a9a  --t3:#5a5a6a
--brd:rgba(255,255,255,0.06)  --brd-a:rgba(79,125,245,0.4)
```

### CSS-паттерны (именование)
При добавлении новых страниц-табов используй существующие паттерны:
- **Hero-блок:** `.{prefix}-hero` (biz-hero, inv-hero, join-hero)
- **Сетка карточек:** `.{prefix}-grid` (biz-grid, inv-grid, tech-grid)
- **Карточка:** `.{prefix}-card` (biz-card, inv-card, tcard)
- **CTA-блок:** `.{prefix}-cta` (biz-cta, inv-cta)
- **Arch/Why-блок:** `.{prefix}-why` или `.tech-arch` (с visual-кругом)
- **Section header:** `.sh` → `.label` + `h2` + `p`
- **Reveal-анимация:** добавлять `.reveal` к блокам

### Responsive breakpoints
| Брейкпоинт | Что меняется |
|---|---|
| `1100px` | Сетки → 2 колонки, навигация уменьшается |
| `1024px` | Сетки → 1 колонка, flex-блоки → column |
| `900px` | Мобильное меню (гамбургер), скрытие навигации |
| `768px` | Минимальные отступы, вертикальная раскладка |
| `480px` | Ультра-компакт: логотип → "LA", минимальные размеры |

### Модальная форма
- Открытие: `openModal()`, закрытие: `closeModal(event)`, Escape
- Поля: имя, Telegram/телефон, тип работы (select с B2C и B2B опциями), комментарий
- Select содержит разделитель `<option disabled>— Для бизнеса —</option>`

## Правовые ограничения (КРИТИЧЕСКИ ВАЖНО)

### ФЗ-383 от 30.10.2018
**Полный запрет на рекламу** услуг по написанию учебных работ.

#### Запрещённые формулировки (B2C)
❌ «написание на заказ», «написать диплом», «заказать работу», «купить», «сделаем за вас», «закажите»

#### Безопасные формулировки (B2C)
✅ «помощь с подготовкой», «помощь в написании», «сопровождение и консультирование», «получить консультацию», «помощь студентам», «поможем подготовить», «узнайте стоимость»

#### B2B — без ограничений
Техническая документация для бизнеса **НЕ подпадает** под ФЗ-383. Можно использовать прямой язык, скидки, акции, любые рекламные каналы.

### ERID-маркировка
С сентября 2022 вся интернет-реклама в РФ требует пометку «Реклама» и ERID. Штраф до 500 000 ₽.

### Дисклеймер
> «Материалы, предоставляемые Logic Architecture, носят вспомогательный характер и предназначены для помощи в подготовке учебных работ.»

## Маркетинговая стратегия

Файл: `docs/strategy/marketing_strategy.md` (v2.1)

### Целевые сегменты
| Сегмент | Ср. чек | Приоритет |
|---|---|---|
| А: Заочники (28-40 лет) | 20-50к | Основной |
| Б: Студенты в стрессе (19-24) | 5-15к | Массовый |
| В: Аспиранты/магистры | 15-50к+ | Премиум |
| Д: Бизнес-клиенты (B2B) | 50-100к | Растущий (с месяца 3) |

### Ценообразование (синхронизировано с сайтом)
**B2C:** Рефераты от 3к, курсовые от 12к, дипломы от 35к, магистерские от 50к, статьи от 15к, оригинальность от 5к
**B2B:** Проектная от 25к, техническая от 20к, тестовая от 15к, юридическая от 10к, Pitch Deck от 35к, полный пакет от 80к

### Каналы
- **Telegram** — основной канал (канал + бот + Telegram Ads)
- **Авито** — листинг B2C и B2B
- **SEO** — органический трафик
- **Habr** — B2B экспертные статьи (с месяца 4)
- **Сарафанное радио** — реферальная программа

## Инфраструктура и деплой

### Хостинг
- **Cloudflare Pages** — статический хостинг для `docs/index.html`
- Publish directory: `docs/`
- Build command: нет (статический сайт)
- Production branch: `main`
- **Автодеплой** при `git push origin main`
- URL: `https://logic-architecture.pages.dev`
- Домен: пока `*.pages.dev`, кастомный домен будет подключен позже

### Cloudflare Worker (бэкенд форм)
- **URL:** `https://la-api.ravinski-genlawyer.workers.dev`
- **Эндпоинты:** `POST /order` (заявка), `POST /join` (присоединение)
- **Конфиг:** `worker/wrangler.toml`
- **Код:** `worker/worker.js`
- **Secrets:** `TG_TOKEN`, `TG_CHAT_ID` (через `wrangler secret put`)
- Формы на сайте → Worker → Telegram Bot API → чат основателя

### Антиспам
- Honeypot-поле (`data.website`) — бот заполняет, обычный пользователь нет
- Rate limit через `localStorage` — 1 заявка в 60 секунд
- Задержка 3 секунды после загрузки страницы

### Инструменты Claude
- Claude может управлять Cloudflare Dashboard через браузер (Chrome MCP + Windows MCP)
- Claude может деплоить Worker через `wrangler deploy` (bash)

## Конвенции разработки

### При редактировании index.html
1. **Не разбивать на файлы** — проект осознанно сделан single-file
2. **Минифицировать CSS** — стили написаны компактно, без лишних пробелов
3. **Сохранять порядок секций** — CSS комментарии `/* ===== SECTION ===== */` разделяют блоки
4. **Добавлять `.reveal`** к блокам, которые должны анимироваться при скролле
5. **iOS Safari** — `inset` всегда дублировать через `top/right/bottom/left`, `-webkit-backdrop-filter` параллельно с `backdrop-filter`
6. **Responsive** — при добавлении новых классов добавлять правила во все нужные брейкпоинты

### При редактировании стратегии
1. Все формулировки проверять по справочнику безопасных фраз (раздел 1.2 стратегии)
2. Цены должны быть синхронизированы между `docs/index.html` и `docs/strategy/marketing_strategy.md`
3. B2B-материалы маркировать отдельно, указывая что ФЗ-383 не применяется

### Работа с файлами на Windows
- Использовать прямые слэши (`C:/Courses/docs/`) в bash-командах
- Пути с кириллицей и пробелами оборачивать в двойные кавычки
- Не использовать backslash (`\`) в mv/cp/rm командах — приводит к созданию мусорных файлов

### Общие правила
- Язык интерфейса: **русский**
- Тон: **экспертный, уверенный, поддерживающий** (не агрессивный, не панибратский)
- Никогда не использовать «DAO» в клиентском позиционировании
- Личный бренд основателя: «Ex-Head of Systems Analysis, крупнейшие банки РФ»
- Не использовать: «генерация текстов», «написание нейросетью», «AI-инжиниринг текстов»
- Использовать: «разработал методику подготовки», «технологический подход»
